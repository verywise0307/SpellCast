## 4. 남아 있는 문제

### `/signal`의 `"none"` enum 변환 오류

최신 로그에서도 아래 오류가 반복된다.

```text
LogJson: Error: JsonValueToUProperty - Unable to import enum
from numeric value none for property CallFunc_GetField_OutValue
```

이번 로그에서 동일 문구가 1,426회 기록됐다. 주문이 없는 응답값 `"none"`을 `ESpellID`로 직접 변환하는 흐름이 아직 남아 있는 것으로 보인다.

권장 처리:

1. JSON의 spell 값을 먼저 String으로 읽는다.
2. `"none"`이면 enum 변환과 주문 발동을 건너뛴다.
3. 유효한 주문 문자열일 때만 `ESpellID`로 변환한다.
