SPELLCAST 게임 메커니즘 기획서
===============================

문서 상태
---------
- 확정된 기억과 현재 프로젝트에서 발견한 흔적을 합쳐 복원한 초안이다.
- [확정]은 사용자가 기억하는 기존 기획이다.
- [프로젝트 흔적]은 현재 저장소에서 확인된 내용이다.
- [신규 초안]은 흔적이 없어 새로 설계한 내용이며 추후 플레이 테스트로 조정한다.


1. 게임 개요
------------
- [확정] 장르: 웹캠 손동작 인식을 사용하는 실시간 마법사 PvP 대전 게임
- [확정] 플레이어는 키 입력 대신 실제 손동작으로 마법을 시전한다.
- [확정] 플레이어 캐릭터는 정해진 위치에 고정되며 이동 조작이 없다.
- [확정] 전체 마법 종류는 10개다.
- [확정] 한 경기에서 플레이어가 선택하여 가져가는 마법은 6개다.
- [확정] 선택한 6개 중 현재 펼쳐져 있어 즉시 사용할 수 있는 마법은 3개다.
- [확정] 마법 자원은 최대 10코스트이며, 사용한 마법의 비용만큼 감소한다.
- [확정] 코스트는 일정 시간마다 1씩 자동 회복된다.
- [확정] 양손 마법은 높은 코스트를 요구하지만 강력하다.
- [확정] 한손 마법은 비교적 저렴하고 빠르게 전투 흐름을 만든다.

[신규 초안] 전투의 핵심은 이동이나 에임 실력이 아니라 상대의 활성 마법과 코스트를
읽고 공격, 방어, 회복, 방해 마법을 알맞은 타이밍에 사용하는 심리전이다.


2. 한 경기의 기본 흐름
----------------------
1) 대전 시작 전에 전체 10개 마법 중 6개를 골라 개인 덱을 구성한다.
2) 선택한 6개 중 3개가 현재 사용 가능한 활성 마법 슬롯에 표시된다.
3) 양 플레이어는 서로 마주 보는 고정 위치에서 전투하며 공격 마법은 상대를 자동 조준한다.
4) 웹캠에서 활성 슬롯 중 하나와 대응하는 손동작을 취한다.
5) 같은 손동작이 8프레임 이상 연속 인식되면 시전 요청으로 판정한다.
6) 서버가 현재 코스트, 활성 슬롯, 쿨다운 및 캐릭터 상태를 검증한다.
7) 검증에 성공하면 코스트를 차감하고 마법을 발동한다.
8) 사용한 마법은 덱 순환의 맨 뒤로 이동하고, 대기 중인 다음 마법이 빈 슬롯에 들어온다.
9) 코스트는 정해진 회복 주기마다 1씩 차서 최대 10까지 회복된다.

[신규 초안] 덱 순환은 클래시 로얄식 순환을 따른다. 6개 중 3개가 활성 상태이고,
사용한 마법은 대기열 뒤로 이동한다. 따라서 강한 마법만 연속 사용할 수 없으며
현재 손패와 코스트를 함께 관리해야 한다.


3. 손동작 판정 규칙
-------------------
- 같은 클래스가 연속 8프레임 검출되어야 유효한 시전으로 인정한다.
- 8프레임 도달 전 다른 동작이 잡히거나 손을 놓치면 누적값은 0으로 초기화한다.
- 한 번 확정된 동작은 손을 풀거나 다른 동작으로 바뀔 때까지 재발동하지 않는다.
- 신뢰도가 기준값보다 낮은 프레임은 유효 프레임에 포함하지 않는다.
- 해당 동작의 마법이 현재 활성 3슬롯에 없으면 시전하지 않는다.
- 코스트가 부족하면 시전하지 않고 UI로 부족 상태를 알려준다.
- 클라이언트의 AI 판정은 시전 요청일 뿐이며 실제 발동 여부는 서버가 결정한다.

[신규 초안] 초기 신뢰도 기준값은 0.55로 두고 실제 조명과 카메라 환경에 맞춰 조정한다.


4. 코스트 시스템
----------------
- 최대 코스트: 10 [확정]
- 마법 시전 시 해당 마법의 코스트만큼 즉시 차감 [확정]
- 일정 시간마다 코스트 1 회복 [확정]
- 최대치를 넘겨 저장할 수 없음 [신규 초안]
- 코스트가 부족한 마법의 슬롯은 어둡게 표시 [신규 초안]

[신규 초안/밸런스 시작값]
- 경기 시작 코스트: 5
- 기본 회복 속도: 1.5초마다 1
- 경기 종료 60초 전 가속: 0.75초마다 1

위 세 값은 플레이 테스트 전 임시값이다.


5. 마법 10종 및 HaGRIDv2 손동작 매핑
-------------------------------------

01) 파이어볼 (Fire Ball)
    손동작: fist / 한손
    코스트: 2
    역할: 기본 공격
    효과: 직선으로 빠르게 날아가 단일 대상에게 피해를 준다.
    비고: [프로젝트 흔적] fist -> fire_ball 매핑이 README와 테스트 코드에 존재한다.

02) 윈드 블라스트 (Wind Blast)
    손동작: palm / 한손
    코스트: 2
    역할: 저비용 투사체 방어
    효과: 자신에게 날아오는 가장 가까운 적 투사체 하나를 소멸시킨다.
    즉발형 마법과 메테오는 막을 수 없다.
    비고: [프로젝트 흔적] palm -> wind_blast 매핑이 존재한다.

03) 아이스 스피어 (Ice Spear)
    손동작: peace / 한손
    코스트: 3
    역할: 견제, 코스트 감소
    효과: 빠른 얼음 창을 자동 발사한다. 적중하면 피해를 주고 상대 코스트를 즉시 1 감소시킨다.
    방어막에 막히거나 윈드 블라스트로 제거되면 코스트도 감소하지 않는다.
    비고: [프로젝트 흔적] 기존 v -> ice_spear를 HaGRIDv2의 peace로 정규화했다.

04) 라이트닝 (Lightning)
    손동작: rock / 한손
    코스트: 4
    역할: 즉발 공격, 마무리
    효과: 상대를 자동 지정해 짧은 예고 후 번개를 떨어뜨린다.
    투사체가 아니므로 윈드 블라스트로 지울 수 없지만 아케인 배리어로 방어할 수 있다.
    비고: [프로젝트 흔적] 기존 index -> lightning 매핑을 기획에 따라 rock으로 변경했다.

05) 리커버리 (Recovery)
    손동작: like / 한손
    코스트: 4
    역할: 회복
    효과: 자신의 체력을 고정량만큼 즉시 회복한다. 최대 체력을 넘을 수 없다.
    비고: [프로젝트 흔적] 기존 thumb -> heal을 like -> recovery로 정규화했다.

06) 마나 드레인 (Mana Drain)
    손동작: grip / 한손
    코스트: 4
    역할: 코스트 회수형 공격
    효과: 느린 마력 투사체를 발사한다. 상대에게 적중하면 작은 피해를 주고
    내 코스트를 2 회복한다. 방어막에 막히거나 윈드 블라스트로 제거되면 회복하지 않는다.
    비고: [신규 초안]

07) 메테오 (Meteor)
    손동작: holy / 양손
    코스트: 7
    역할: 강력한 지연 공격, 방벽 파괴
    효과: 상대 위치에 긴 경고 후 운석을 떨어뜨려 큰 피해를 준다.
    아케인 배리어에 추가 피해를 주지만 긴 예고 시간 때문에 상대가 방어를 준비할 수 있다.
    비고: [확정] holy는 메테오이며 7코스트다.

08) 아케인 배리어 (Arcane Barrier)
    손동작: xsign / 양손
    코스트: 6
    역할: 강력한 방어
    효과: 짧은 시간 자신 앞에 대형 방벽을 생성한다. 투사체를 막고 즉발 및 메테오 피해를
    크게 줄이지만, 메테오를 막으면 방벽이 즉시 파괴된다.
    비고: [신규 초안]

09) 하트 생추어리 (Heart Sanctuary)
    손동작: hand_heart / 양손
    코스트: 7
    역할: 지속 회복, 버티기
    효과: 자신의 고정 위치에 일정 시간 유지되는 성역을 만들고 체력을 지속 회복한다.
    시전 중 공격받아도 사라지지 않지만 총 회복량은 즉시 공격 마법의 피해보다 느리다.
    비고: [신규 초안]

10) 마나 가속 (Mana Surge)
    손동작: ok / 한손
    코스트: 3
    역할: 자기 코스트 회복 강화
    효과: 6초 동안 내 코스트 자연 회복량이 2배가 된다.
    기본 회복 주기는 그대로 유지하고, 회복 시 1 대신 2코스트를 얻는다.
    효과 중 다시 사용해도 배율은 중첩되지 않으며 지속시간만 처음부터 다시 계산한다.
    비고: [신규 초안]


6. 마법 구성 의도
-----------------
- 저비용 한손 마법 3개: 파이어볼, 윈드 블라스트, 아이스 스피어
- 중비용 한손 마법 3개: 라이트닝, 리커버리, 마나 드레인
- 코스트 강화 한손 마법 1개: 마나 가속
- 고비용 양손 마법 3개: 메테오, 아케인 배리어, 하트 생추어리
- 공격, 방어, 회복, 제어가 모두 존재해 6개 덱 구성에 선택지가 생긴다.
- 이동과 수동 조준이 없으므로 투사체 여부, 방어 타이밍, HP와 코스트 교환이 상성의 핵심이다.
- 모든 마법은 투사체, 방어막, 상대 HP, 내 HP, 양쪽 코스트 중 하나 이상에만 간섭한다.
- 활성 슬롯 봉인, 이동 방해, 조작 불능처럼 별도 상태 시스템이 필요한 효과는 사용하지 않는다.
- 양손 마법은 손을 크게 사용해 시전 의도가 잘 드러나며 상대에게 대응할 시간을 준다.
- 손동작은 모두 공식 HaGRIDv2 클래스 이름을 그대로 사용한다.

8. PvP 서버 판정 원칙
---------------------
- detector는 웹캠 영상과 손동작을 로컬 PC에서 처리한다.
- 서버에는 원본 영상이 아니라 확정된 gesture와 spell 요청만 전송한다.
- 서버는 플레이어별 현재 코스트와 활성 3슬롯을 보관한다.
- 서버가 코스트 차감, 덱 순환, 피해, 회복, 투사체와 방어막을 최종 판정한다.
- 서버가 모든 공격의 대상을 상대 플레이어로 자동 지정한다.
- Unreal 클라이언트는 서버가 승인한 결과를 받아 애니메이션과 이펙트를 재생한다.
- 클라이언트가 임의 spell 문자열을 보내도 활성 슬롯에 없거나 코스트가 부족하면 거부한다.


9. 현재 프로젝트에서 확인된 기존 흔적
--------------------------------------
- fist -> fire_ball
- v -> ice_spear
- palm -> wind_blast
- thumb -> heal
- index -> lightning
- both_hands -> ultimate

위 매핑은 초기 HTTP 통신 테스트용으로만 존재하며 코스트나 구체 효과는 구현되어 있지 않다.
현재 기획에서는 실제 HaGRIDv2 클래스명에 맞게 v를 peace, thumb를 like,
index를 rock으로 바꾸고, 모호한 both_hands 대신 구체적인 양손 클래스 3개를 사용한다.


10. 추후 확정할 항목
--------------------
- 플레이어 최대 체력과 각 마법의 정확한 피해량/회복량
- 기본 코스트 회복 주기와 경기 후반 가속 여부
- 마법별 개별 쿨다운 사용 여부
- 경기 제한 시간, 승리 조건, 연장전 규칙
- 활성 3슬롯의 순환 순서가 고정인지 무작위인지
- 손동작 오인식 시 취소 UI와 사용자별 신뢰도 보정 방식
- 양손 마법의 시전 시간과 시전 중 피격 시 취소 여부
- 마나 가속의 지속시간과 코스트
- 윈드 블라스트가 제거할 수 있는 투사체의 세부 목록
