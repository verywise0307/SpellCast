## 2. 오늘 대화와 마나 오류 조사

### 최초 증상

- 마법을 한 번 커밋했는데 마나가 여러 번 감소한 것처럼 보였다.
- 이전 PIE 로그에서는 마나가 `8`에서 `-15`로 내려가는 구간과 동일한 숫자가 6회 또는 12회 출력되는 구간이 있었다.
- 처음에는 활성 주문 배열 전체의 비용이 차감된 것으로 의심했으나, Blueprint 화면을 확인한 뒤 이 추정은 철회했다.

### Blueprint 화면으로 확인한 실제 구조

`BP_SpellCastPlayerState.CommitCast`:

1. `Get Spell Cost(Spell ID)`
2. 기존 Mana를 보관
3. `Mana - Spell Cost` 계산 후 `Set Mana`
4. `SpellCycle`에서 사용 주문 제거 후 다시 추가
5. `For Each SpellCycle -> Enum to String -> Print String`

따라서 한 커밋 직후 주문 이름 여섯 개가 출력되는 것은 `SpellCycle` 배열을 디버그 출력한 것이며, 여섯 주문의 비용을 모두 차감했다는 뜻이 아니다.

이후 UI 갱신 구간:

- `Get All Widgets of Class(WBP_HoldingSpell) -> For Each`
- 각 HoldingSpell 위젯에서 사용 애니메이션 실행
- 루프 내부에서 `ManaPercent` 계산 및 `Get All Widgets of Class(WBP_playerstate/mpbar)`를 다시 실행

동일한 Mana 숫자가 여러 번 보이는 현상은 위젯 수만큼 UI 갱신/Print가 반복되는 구조의 영향일 가능성이 높다. 제공된 화면 범위에서는 실제 `Set Mana`는 루프 앞에 있어 한 번만 실행된다.

### Get Spell Cost 확인

`Get Spell Cost` 함수:

- `Spell ID -> Enum to String -> Name`
- `Get Data Table Row(DT_SpellData)`
- Row Found: 해당 행의 `Cost` 반환
- Row Not Found: `999` 반환

`DT_SpellData` 비용:

| SpellID | Cost |
|---|---:|
| FireBall | 2 |
| WindBlast | 2 |
| IceSpear | 3 |
| Lightning | 4 |
| Recovery | 4 |
| ManaDrain | 4 |
| Meteor | 7 |
| ArcaneBarrier | 6 |

테이블 값은 기획과 일치했다. `Row Not Found = 999`는 위험하므로 추후 실패 시 커밋 자체를 중단하는 구조로 바꾸는 것이 안전하다.

## 4. 남아 있는 확정 문제

### A. 마나 부족인데 마법 실행 로그가 발생

다음 주문은 `BP_witch` 시전 로그가 있으나 대응하는 `commit mana` 로그가 없다.

- 17:08:19 전후 `HeartSanctuary`
- 17:09:01 전후 `Meteor`

비용 검증에서 커밋은 거부됐지만, 호출부의 마법 발동 실행선은 계속 진행했을 가능성이 높다.

권장 구조:

```text
CanAffordSpell?
├─ False -> 부족 UI/로그 후 종료
└─ True
   -> CommitCast
   -> Spawn/Activate Spell
```

가능하면 `CommitCast`가 성공 여부를 Boolean으로 반환하고, 성공한 경우에만 `BP_witch`의 실제 주문 발동을 실행한다.

### B. `/signal`의 `"none"` enum 변환 오류

최신 로그에서도 아래 오류가 총 424회 반복됐다.

```text
LogJson: Error: JsonValueToUProperty - Unable to import enum
from numeric value none for property CallFunc_GetField_OutValue
```

로컬 브리지 `/signal`은 주문이 없을 때 문자열 `"none"`을 반환한다. Unreal 쪽에서 이를 `ESpellID`로 직접 변환하면서 실패하는 것으로 보인다.

권장 수정:

- JSON의 spell을 먼저 String으로 읽는다.
- `"none"`이면 enum 변환과 커밋을 건너뛴다.
- 그 외 문자열만 `ESpellID`로 변환한다.
- 또는 `ESpellID`에 명시적인 `None` 값을 추가하고 브리지 반환 문자열과 정확히 맞춘다.

### C. UI 갱신 중첩 루프

`ManaPercent` 계산과 MP Progress Bar 검색이 HoldingSpell 루프의 `Loop Body` 뒤에 있어 위젯 개수만큼 반복된다.

권장 수정:

- HoldingSpell 애니메이션 루프는 그대로 둔다.
- Mana UI 갱신은 바깥 루프의 `Completed`에서 한 번만 호출한다.
- 더 나은 구조는 PlayerState가 Mana만 복제하고, 소유 클라이언트 UI가 RepNotify/이벤트로 자체 갱신하는 것이다.

### D. 마나 하한과 Data Table 조회 실패 처리

- 차감 전 `Mana >= SpellCost` 검사를 서버 권한에서 수행한다.
- 차감 결과는 `Clamp(Mana - SpellCost, 0, MaxMana)`로 제한한다.
- Data Table Row Not Found에서 `999`를 비용으로 반환하지 말고 실패 처리 후 커밋을 중단한다.
