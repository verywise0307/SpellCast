# SpellCast 작업 인수인계 — 2026-07-20

## 오늘 요청 및 수정한 내용

손동작 탐지 모델에서 같은 의미의 복수 클래스를 하나의 마법으로 처리하도록 Python 매핑을 수정했다.

실제 `detect/hagrid-v2-yolov10n.pt`의 메타데이터에서 확인한 관련 클래스명:

```text
hand_heart
hand_heart2
palm
stop
```

현재 의도한 매핑:

```text
hand_heart  -> HeartSanctuary
hand_heart2 -> HeartSanctuary
heart       -> HeartSanctuary  # 호환용 별칭
heart2      -> HeartSanctuary  # 호환용 별칭

palm        -> WindBlast
stop        -> WindBlast
```

수정 파일:

- `client_bridge/main.py`
  - `SPELL_BY_GESTURE`에 `stop`, `hand_heart2`, `heart`, `heart2` 추가
- `detect/main.py`
  - `GESTURE_LABELS`에 `stop`, `hand_heart2`, `heart`, `heart2` 추가

주의: 실행 중인 detector와 client bridge를 완전히 종료하고 다시 실행해야 Python 변경이 반영된다.

## Python 실행 환경 문제

시스템에서 `python`과 `py` 실행기를 찾지 못했다. `detect/.venv/Scripts/python.exe`는 존재하지만, 내부적으로 다음의 삭제되었거나 이동된 Python을 가리켜 실행되지 않는다.

```text
C:\Users\M\AppData\Local\Programs\Python\Python311\python.exe
```

따라서 Python AST 문법 검사는 실행하지 못했다. 변경분은 단순 딕셔너리 항목 추가이며 `git diff --check`는 통과했다. 다음 작업자는 필요하면 Python 설치 또는 `detect/.venv` 재생성을 먼저 처리할 것.

## 현재 해결되지 않은 Unreal UI 문제

사용자는 카드 선택 후 다음 세 위젯을 순차 생성하고 Viewport에 추가하도록 `WBP_choosecards`를 구성했다.

```text
WBP_Webcam
WBP_Holdingspell
WBP_Playerstate
```

증상:

- PIE에서는 앞의 `WBP_Webcam`, `WBP_Holdingspell` 두 개만 보인다.
- `WBP_Playerstate`의 `Event Construct`에 Print String을 추가해도 출력되지 않았다.
- 더 중요한 확인 결과: `WBP_choosecards`에서 세 UI 생성 노드를 전부 끊어도 PIE에서는 앞의 두 위젯이 계속 나타난다.
- 즉, 현재 편집 중인 `WBP_choosecards`의 변경이 실제 PIE 실행 경로에 반영되지 않거나, 앞의 두 위젯이 다른 Blueprint 경로에서 별도로 생성되고 있다.

다음 작업자가 우선 확인할 것:

1. PIE를 완전히 중단한 뒤 모든 Blueprint를 Compile/Save하고 에디터를 재시작한다.
2. 실제 카드 선택 화면을 만드는 `Create Widget` 노드의 Class가 지금 편집 중인 정확한 경로의 `WBP_choosecards`인지 확인한다. 같은 이름의 복사본, 자식 클래스, 오래된 클래스 참조 가능성을 확인한다.
3. Content Browser의 `Find References` 또는 Blueprint 전체 검색으로 아래 생성 위치를 모두 찾는다.

```text
Create WBP Webcam Widget
Create WBP Holdingspell Widget
Create Widget
```

4. 특히 `BP_playercontroller`, HUD, GameMode, Level Blueprint, Character/Pawn, GameInstance의 BeginPlay/UI 초기화 경로를 확인한다.
5. `WBP_choosecards.Event Construct` 첫 줄에 고유 문자열을 출력해 실제 인스턴스가 생성되는지 확인한다.
6. Widget Reflector를 사용할 수 있으면 PIE에서 표시된 Webcam/Holdingspell 위젯의 실제 타입과 경로를 확인한다.

현재 정황상 단순 ZOrder/Visibility 문제가 아니라 **다른 생성 경로 또는 다른 WBP_choosecards 클래스가 실행되는 문제**가 가장 유력하다.

## 관련 Blueprint 런타임 오류

오늘 다음 오류도 보고되었다.

```text
CallFunc_Array_Get_Item 프로퍼티를 읽으려던 중 None에 접근했습니다.
Node: Print String
Graph: EventGraph
Function: Execute Ubergraph BP Playercontroller
Blueprint: BP_playercontroller
```

이는 `BP_playercontroller`에서 배열 `Get` 결과가 `None`인 상태로 사용된 것이다. UI가 다른 경로에서 생성되는 문제와 연관됐을 가능성이 있으므로 `BP_playercontroller`의 UI 참조 배열, 인덱스 유효성, 빈 슬롯을 함께 확인할 것.
